# A11Y (Tier-0)

## 1. 목표

- WCAG 2.2 AA 수준을 지향(구현 난이도 고려하여 단계적 적용).
- 키보드 내비게이션/포커스 가시성/스크린리더 호환성 보장.

## 2. 일반 원칙

- 모든 상호작용 요소: `role`, `aria-*`, 키보드 핸들러 제공.
- 포커스 순서와 트랩 제어(모달/팝오버).
- 대비: 텍스트 대비 비율 4.5:1 이상(아이콘 3:1 이상 권장).

## 3. 컴포넌트 체크리스트(초안)

- Button: `aria-pressed`(토글형), `disabled` vs `aria-disabled` 구분.
- Dialog: `role="dialog"`, `aria-modal="true"`, 초기 포커스/포커스 트랩/ESC 종료.
- Listbox/Combobox: `aria-activedescendant`, `aria-expanded`, `aria-controls`.
- Tooltip: 비필수 정보 → `aria-describedby`, 포커스 가능 요소만 트리거.

## 4. 테스트

- 스크린리더(Windows NVDA/JAWS, macOS VoiceOver) 간단 스모크.
- 키보드 탭 순서/포커스 링 수동 확인.

## 5. 문서화

- 컴포넌트별 A11Y 섹션 필수: 역할/키보드 표/예시 코드.
